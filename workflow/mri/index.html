Quality Control Workflow

Where quality control fits in the pipeline, and what to look for at each inspection point. Raw data is inspected before any processing is run; processed data is inspected again before it enters analysis. Both inspections use the checklists below.

Pipeline

1

Data acquisition

Scanning, DICOM export, conversion to NIfTI/BIDS.

2

Preprocessing

Realignment, coregistration, normalization, distortion correction, smoothing.

3

Analysis

Modelling, group statistics, reporting.

Inspection point A

Inspect raw data

Before anything is processed. Exclusion criteria should already be written down at this stage.

Structural checklist Functional checklist

Inspection point B

Inspect processed data

After preprocessing, before the data enter any model. Failures here are usually fixable by rerunning with different settings.

Structural checklist Functional checklist

An inspection is not a gate you pass once. If preprocessing is rerun with different settings, inspection point B is repeated. If a scan fails inspection point A, note the reason against the pre-registered exclusion criteria rather than deciding case by case.

Checklists

Tick items as you review each scan. Items marked Examples link to an artifact page with annotated images — open it alongside the scan if you are unsure what you are looking at. Nothing is saved between page loads, so use Copy report to paste the result into your QC log before you move on.
